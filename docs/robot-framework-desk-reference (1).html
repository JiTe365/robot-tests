2) Architecture / dependencies (repo & tools)

flowchart TB WS["WS: Windows/Linux Shell — runs testrunner or CI"] BAT["BAT: testrunner.bat (local) / CI step — orchestrates Pipenv"] subgraph RT["RT: Project Root (C:/robot-tests)"] PF["PF: Pipfile — dependency spec"] SUITE["SUITE: tests/*.robot — your test suites"] INIT["INIT: tests/__init__.robot — shared setup/teardown"] RES["RES: results/ — log.html, report.html"] end subgraph PE["PE: Pipenv Virtualenv (.venv)"] PIPENV["PIPENV: pipenv — resolves & installs"] PKG["PKG: Packages — robotframework, robotframework-browser, requests"] CMD_INIT["CMD1: pipenv run python -m Browser.entry init — runs inside the venv"] CMD_ROBOT["CMD2: pipenv run robot -d results tests — runs inside the venv"] DL["DL: Browser.entry init — downloads Playwright browsers"] end subgraph PW["PW: Playwright Runtime"] DRV["DRV: Playwright driver"] BROW["BROW: Chromium/Firefox/WebKit"] end WEB["WEB: System Under Test (e.g., https://playwright.dev/)"] WS --> BAT BAT -->|sync/install| PIPENV PIPENV -->|reads| PF PIPENV -->|installs| PKG BAT --> CMD_INIT CMD_INIT --> DL DL --> DRV BAT --> CMD_ROBOT CMD_ROBOT --> SUITE INIT --> SUITE SUITE -->|imports| PKG SUITE -->|calls keywords| DRV DRV -->|launch| BROW BROW -->|navigate| WEB SUITE --> RES

Note: Both pipenv run python -m Browser.entry init and pipenv run robot -d results tests execute inside the Pipenv virtualenv (PE). That keeps imports consistent.

Legends (abbreviations)

  • WS — Shell (PowerShell/CMD/Bash)
  • BAT — local batch script or CI step that invokes pipenv
  • PIP / PIPENV / PE — pipenv virtual environment/command
  • PF — Pipfile (dependency spec)
  • PKG — installed packages (e.g., robotframework, robotframework-browser)
  • CMD1 — pipenv run python -m Browser.entry init (runs inside the venv)
  • CMD2 — pipenv run robot -d results tests (runs inside the venv)
  • DL — Browser.entry init step (downloads Playwright browsers)
  • PW — Playwright driver (used by Browser library)
  • BR / BROW — actual browser process (Chromium/Firefox/WebKit)
  • WEB / SUT — System Under Test (your app/site)
  • RES — results directory (log.html, report.html)
  • ST — *** Settings *** section or testsuite.robot in sequence diag
  • SV — *** Variables *** section
  • SK — *** Keywords *** section (custom keywords)
  • SC — *** Test Cases *** section
  • L — Libraries (e.g., Browser)
  • R — Resource files (.robot/.resource)
  • RK — Keywords from resource files
  • RV — Variables from resource files
  • VF — Variable files (.py/.yaml/.robot)
  • LK — Keywords provided by libraries (e.g., New Page, Get Title)
  • INIT — __init__.robot (directory-level suite config)
